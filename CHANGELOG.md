# Changelog

All notable changes to **Terraform Healer** will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added
- 🚀 Initial public release as **Terraform Healer**
- 🤖 AI-powered Terraform error detection and remediation
- 🎯 8-layer validation pipeline for safety
- 🔒 Pattern-based risk classification (LOW/MEDIUM/HIGH/CRITICAL)
- 📝 Rich GitHub issue creation with detailed metadata
- 🔀 Automated pull request creation with risk assessment
- 💬 Slack integration with emoji-rich notifications
- 🌐 Multi-organization and multi-repository support
- 🔐 Dual authentication: GitHub App and Personal Access Token
- ☁️ Kubernetes deployment with health checks and resource limits
- 🐳 Multi-stage Docker build with Terraform tooling
- 🧪 Comprehensive test suite
- 📚 Extensive documentation (8+ guides)
- 🎨 100% emoji coverage for better UX
- 🔧 Static Terraform validation without cloud credentials
- ✅ Support for reusable workflows from 3rd party orgs
- 🎭 Multi-AI agent validation (generate, review, security check)
- 📊 Structured logging with contextual information
- 🔄 GitHub Actions for CI/CD and Docker builds
- 🛡️ Webhook signature verification
- 📦 Change limits: max 2 files, 20 lines per file
- 🚫 Protected files: backend.tf, .tfstate, .terraform.lock.hcl

### Documentation
- 📖 Comprehensive README with quick start
- 🤝 Contributing guidelines
- 📜 Code of Conduct
- 🔒 Security policy
- 🚀 Deployment guide with 8-week rollout
- 😀 Emoji usage guide
- 🔐 GitHub authentication guide
- 📝 GitHub issue best practices
- 🎯 Issue examples and templates
- 🔄 Reusable workflows documentation
- 💾 Database architecture explanation
- 🗺️ Future roadmap with 18 planned features

### Infrastructure
- ☸️ Kubernetes manifests (Deployment, Service, ConfigMap, Secrets)
- 🐳 Dockerfile with Terraform 1.5, tflint, tfsec
- 🏗️ GitHub Actions for Docker build and push to GHCR
- 🧪 CI workflow for tests and linting
- 📦 Requirements management with exact versions
- 🔧 Build script for local Docker builds
